# Trackarr Tracker (Go)

Standalone BitTorrent HTTP tracker. Built in Go, ships as a ~13 MB
`scratch`-based Docker image, and uses [sqlc](https://sqlc.dev/) for
type-safe database access.

## Running locally

```bash
DATABASE_URL=postgres://tracker:tracker@localhost:5432/trackarr \
REDIS_URL=redis://localhost:6379 \
REDIS_PASSWORD=... \
IP_HASH_SECRET=$(openssl rand -hex 32) \
go run ./cmd/tracker
```

## Adding a new query

1. Edit `db/queries/*.sql` and add the annotated query:
   ```sql
   -- name: GetTorrentSize :one
   SELECT size FROM torrents WHERE info_hash = $1;
   ```

2. Regenerate the typed Go bindings:
   ```bash
   ./apps/tracker/scripts/gen.sh
   ```

3. Use it in your code:
   ```go
   size, err := s.db.Q.GetTorrentSize(ctx, infoHashHex)
   ```

If sqlc complains your query references an unknown column, refresh
`db/schema.sql` first:
```bash
./apps/tracker/scripts/dump-schema.sh
```

## Schema ownership

Migrations are owned by Drizzle in `packages/db/src/schema.ts`. The tracker
only keeps a **snapshot** of the columns it uses (`db/schema.sql`) so sqlc
can typecheck queries.

When the api-side schema changes:

1. Apply the Drizzle migration as usual.
2. Run `./scripts/dump-schema.sh` to refresh the tracker's snapshot.
3. Update any tracker query that touches the changed columns.
4. `./scripts/gen.sh` to regenerate.
5. The Go compiler will point you at every call site that needs an update.
